Introduction à Git

Git est un système de contrôle de version distribué créé par Linus Torvalds en 2005.

Il permet de suivre l'historique des modifications d'un projet, de collaborer efficacement en équipe, et de gérer différentes versions de votre code.

[Projet] → [Git] → [Historique complet]
[Équipe] ←→ [Collaboration] ←→ [Dépôt distant]

2 / 14

Pourquoi utiliser Git ?

  • Suivre l'historique complet des modifications
  • Collaborer efficacement en équipe
  • Revenir en arrière facilement en cas d'erreur
  • Expérimenter sans risquer le code principal
  • Sauvegarder votre travail sur des dépôts distants
Erreur → [git revert] → Retour en arrière
Expérimentation → [Branche] → Sécurité

Git vs GitHub/GitLab

Git : outil local installé sur votre machine pour gérer les versions.

GitHub/GitLab : plateformes en ligne pour héberger vos dépôts Git et faciliter la collaboration.

[Votre Ordinateur] ←→ [Git] ←→ [GitHub/GitLab]
Local ←→ Synchronisation ←→ Cloud

Configuration Initiale

Configurez votre identité Git une fois pour toutes :

git config --global user.name "Votre Nom"

git config --global user.email "votre@email.com"

[Git Global Config]
├── user.name = "Votre Nom"
├── user.email = "votre@email.com"
└── core.editor = "code"

Créer un Dépôt Local

Initialiser un nouveau dépôt dans un dossier existant :

git init

Cloner un dépôt distant existant :

git clone https://github.com/utilisateur/projet.git

[Dossier vide] → git init → [.git/]
[GitHub] → git clone → [Dossier local + .git/]

Cycle de Base de Git

1. Modifier fichiers → [Espace de travail]
2. git add → [Zone de staging]
3. git commit → [Historique]
4. git push → [Dépôt distant]

Le staging area permet de choisir précisément ce qui sera inclus dans le commit.

Commandes Essentielles

  • git status - État du dépôt
  • git log - Historique des commits
  • git diff - Différences non stagées
  • git checkout - Changer de branche/fichier
  • git reset - Annuler des modifications
[git status] → Affiche les 3 zones

Gestion des Branches

main: A──B──C──D
↑
feature: E──F

Les branches permettent de travailler sur des fonctionnalités indépendamment.

git checkout -b feature-login

Fusionner des Branches

main: A──B──C──D───────G
↑ ↗
feature: E──F

Le commit G est le résultat de la fusion (merge) des branches.

En cas de conflits, résolution manuelle nécessaire.

Dépôts Distants

origin/main: A──B──C
↑
local/main: A──B──C──D──E

git push origin main

git pull origin main

La synchronisation maintient la cohérence entre local et distant.

Bonnes Pratiques

  • Écrire des messages de commit clairs
  • Créer des branches pour chaque fonctionnalité
  • Faire des commits atomiques
  • Ne jamais pousser du code cassé sur main
  • Utiliser .gitignore pour les fichiers temporaires
[main] ← Sécurisée et stable
[feature/*] ← Expérimentation

Résolution de Conflits

main: A──B──C──D
↗ ↖
feature: E──F
↓
[CONFLIT] ← Les commits D et F modifient les mêmes lignes

Éditez manuellement les fichiers, puis git add et git commit.
